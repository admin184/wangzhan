## 三、安装并配置 MariaDB

```bash
# 安装 MariaDB
dnf install -y mariadb-server mariadb

# 启动并设置开机自启
systemctl start mariadb
systemctl enable mariadb

# 安全初始化（设置 root 密码、移除匿名用户等）
# 交互式操作，按提示回答：
#   Enter current password for root: 直接回车（首次无密码）
#   Switch to unix_socket? → n
#   Change root password? → Y，设置密码（牢记）
#   Remove anonymous users? → Y
#   Disallow root login remotely? → Y
#   Remove test database? → Y
#   Reload privilege tables? → Y
mysql_secure_installation
```

> **如果 `mysql_secure_installation` 中设置了 root 密码，记得同步修改 `includes/db.php`。**

## 四、部署项目代码

```bash
# 创建 Web 目录
mkdir -p /var/www/wangzhan

# 从宿主机 / 网络上传代码到 /var/www/wangzhan/
# 这里假设代码已通过 scp / VirtualBox 共享文件夹 / git clone 等方式就位

# 设置目录权限（Apache 运行用户为 apache）
chown -R apache:apache /var/www/wangzhan
chmod -R 755 /var/www/wangzhan

# 确保 Session 存储目录可写
chown apache:apache /var/lib/php/session
chmod 700 /var/lib/php/session
```

## 五、配置 PHP-FPM

编辑 `/etc/php-fpm.d/www.conf`：

```bash
cp /etc/php-fpm.d/www.conf /etc/php-fpm.d/www.conf.bak
vim /etc/php-fpm.d/www.conf
```

修改以下行：

```ini
; Rocky 9 默认即为 apache，通常无需改动；检查确认即可
user = apache
group = apache

; 监听方式改为 Unix socket（性能更好）
listen = /run/php-fpm/www.sock

; 设置 socket 权限
listen.owner = apache
listen.group = apache
listen.mode = 0660
```

启动并验证：

```bash
systemctl start php-fpm
systemctl enable php-fpm
systemctl status php-fpm
```

## 六、安装并配置 Apache (httpd)

```bash
# 安装 Apache
dnf install -y httpd
```

### 6.1 创建虚拟主机配置

```bash
vim /etc/httpd/conf.d/wangzhan.conf
```

写入以下内容：

```apache
<VirtualHost *:80>
    ServerName 192.168.131.5                    # 替换为实际域名或服务器 IP
    DocumentRoot /var/www/wangzhan
    DirectoryIndex index.php index.html

    <Directory /var/www/wangzhan>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>

    # PHP 文件通过 PHP-FPM (Unix socket) 处理
    <FilesMatch \.php$>
        SetHandler "proxy:unix:/run/php-fpm/www.sock|fcgi://localhost"
    </FilesMatch>

    # 禁止访问 include 目录
    <Directory /var/www/wangzhan/includes>
        Require all denied
    </Directory>

    # 禁止访问 .sql 文件
    <FilesMatch "\.sql$">
        Require all denied
    </FilesMatch>

    # 静态资源缓存
    <Directory /var/www/wangzhan/assets>
        <IfModule mod_expires.c>
            ExpiresActive On
            ExpiresByType text/css "access plus 7 days"
            ExpiresByType image/* "access plus 7 days"
        </IfModule>
    </Directory>
</VirtualHost>
```

> **注意**：Apache 默认会加载 `/etc/httpd/conf.d/welcome.conf` 欢迎页，部署后建议注释或删除它：
> ```bash
> mv /etc/httpd/conf.d/welcome.conf /etc/httpd/conf.d/welcome.conf.disabled
> ```

### 6.2 测试并启动 Apache

```bash
# 检查配置语法
httpd -t

# 启动 Apache
systemctl start httpd
systemctl enable httpd
systemctl status httpd
```

### 7.1 修改数据库连接配置

如果 `mysql_secure_installation` 中设置了 root 密码，务必同步修改：

```bash
vim /var/www/wangzhan/includes/db.php
```

```php
// 将空密码改为实际密码
$db_pass = '你设置的密码';
```

> **安全建议**：生产环境不要用 root 连接。创建一个专用数据库用户：
>
> ```sql
> CREATE USER 'wangzhan'@'localhost' IDENTIFIED BY '强密码';
> GRANT SELECT, INSERT ON wangzhan.* TO 'wangzhan'@'localhost';
> GRANT SELECT, INSERT ON wangzhan.comments TO 'wangzhan'@'localhost';
> FLUSH PRIVILEGES;
> ```
>
> 然后修改 `db.php` 中的 `$db_user` 和 `$db_pass`。

## 九、SELinux 配置

Rocky 9 默认启用 SELinux，Apache 访问 `/var/www/` 可能被阻止：

```bash
# 设置文件上下文
semanage fcontext -a -t httpd_sys_content_t "/var/www/wangzhan(/.*)?"
restorecon -Rv /var/www/wangzhan

# 允许 Apache 通过 proxy 模块连接 PHP-FPM socket
setsebool -P httpd_can_network_relay on
setsebool -P httpd_can_network_connect_db on

# 仅排查时临时放宽（不推荐长期使用）
# setenforce 0
```

## 附录：故障排查

| 问题 | 排查命令 |
|------|---------|
| 503 Service Unavailable | `systemctl status php-fpm`；检查 `/run/php-fpm/www.sock` 是否存在 |
| 403 Forbidden | `ls -laZ /var/www/wangzhan` 检查 SELinux 上下文；`tail -f /var/log/httpd/error_log` |
| 数据库连接失败 | `mysql -u root -p wangzhan` 测试；检查 `db.php` 中密码和端口 |
| 样式不加载 | CDN 外网可达性（需服务器能访问 jsdelivr.net）；检查浏览器 Console |
| 登录后 Session 丢失 | 检查 `/var/lib/php/session` 权限是否为 apache:apache |
| SELinux 阻断 | `ausearch -m avc -ts recent \| audit2why` 查看原因 |
| Apache 启动失败 | `journalctl -xeu httpd` 查看详细日志 |
